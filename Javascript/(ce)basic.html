1. About javascript

2. Use Strict & Variables

Modern mode & 주석

            javascript의 주석은 //로 표기된다. 여러줄 주석은 /**/
            use strict

            use strict는 현대적인 방식으로 스크립트가 작동하게 한다. 
            이것은 스크립트 가장 위에 선언해야 한다.
            하지않아도 상관 없지만 use strict를 선언했을 때 최신 javascript는 클래스와 모듈을 지원한다. 
            따라서 따론 선언해줄 필요 없다.

변수

            대부분 javaScript는 imformation과 함께 작동한다. 
            예를들어 온라인 상점에서 판매중인 상품이나 장바구니가 될 수 있고 채팅 application 같은 경우에는
            사용자 또는 메시지등 이라고 할 수 있다.

            let
            변수는 데이터에 대한 지정된 저장소이다. 변수를 만들려면 let을 사용한다.
            let을 사용하여 변수를 선언하면 관련된 메모리 영역에 저장된다.
            let을 사용한 변수 선언은 바꿀 수 있다. 값이 변경되면 이전 데이터가 변수에서 제거된다.
            javascript 같은 경우는 java와 같이 camelCase가 적용된다.

            Const
            만약 불변의 변수를 선언하고 싶다면 const를 사용하면 된다.
            const를 사용하면 재지정할 수 없다. 이미 대중적으로 알려진 값이나 고정된 값은 모두 대문자로 표기하여 선언한다.

3. Data Type

데이터 유형

            Javascript의 값은 8가지 유형이 있다.
            
            숫자 
            - 곱하기, 더하기, 나누기, 빼기 Infinity
            - 큰 정수값은 정확도가 떨어진다. 고정된 6비트 저장소에 맞지 않으면 오류가 발생한다.
            - 최근에 BigInt type은 임의 길이의 정수를 나탄기 위해 최근에 언어가 추가되었다.
            

            문자열
            - 문자열은 따옴표로 묶어야한다.
            - 큰따옴표, 작은 따옴표, 백틱이 있다. " " ' ' 
            - 백틱은 확장된 기능 인용 부호 이다. 변수와 표현식을 래핑하여 문자열에 embed할 수 있다.

            boolean(논리 유형)
            - 부울 유형에는 true,false가 있다.

            Null 값
            - null은 존재하지 않는 객체에 대한 참조 또는 널 포인터가 아니다. 
            - null은 javascript에서 업음, 비어있음(0)을 나타내는 뜻일 뿐이다.
            - 즉 단일 값을 갖는 독립 실행형 유형이다.

            undefined 값
            - 값이 할당되지 않음을 나타낸다.
            - 선언되었지만 할당되지 않은 경우
            - 값이 0을 의미하지 않고 NaN을 의미
            - 일반적으로는 권장하지 않는다.
            
            개체 및 기호
            - object유형은 특별하다. 객체는 데이터 컬렉션과 더 복잡한 entity를 저장하는데 사용된다.

            typeof 연산자와
            - typeof 연산자는 피연산자의 유형을 반환한다. 
            - 다른 유형의 값을 다르게 처리하거나 빠른 확인을 원할 대 유용하다.
            - typeof는 함수가 아니라 연산자이다. 원래 typeof(X)이지만, 여기 괄호가 함수의 괄호가 아니다.

4. Interaction

interactive - 경고, prompt, 확인

            1. alert
            모달창을 통해서 방문자가 창을 처리할 때까지 페이지의 나머지 부분과 상호작용하거나 다음 버튼으 누르는 작업을 할 수 없음
        
            2.prompt   
            두가지 인수를 허용한다.
            텍스트 메시지 방문자를 위한 입력 필드 확인 취소 버튼이 있는 모달창을 보여준다.
            prompt(title,[default])에서 title은 방문자에게 표시할 텍스트 default는 매개변수, 입력 필드 초기값
            []부분은 선택사항이나 값이 없는 경우, undefined를 삽입해주는 것이 좋다.
        
            3.confirm
            이 함수는 확인 및 취소라는 두개의 버튼이 있는 모달창을 보여준다.
            결과는 true이거나 false인 것이다.

5. Type conversions

유형 변환

            1. 문자열 변환
            문자열 형식이 필요할 때 발생. string(value)값을 문자열로 반환하는 함수를 호출할 수 있다.
        
            2. 숫자 변환
            수학 함수 및 표현식의 숫자 변환은 자동적으로 발생한다.
            예를 들어 나눗셈이 /이 숫자가 아닌 것에 적용되는 경우
            문자 타입인 숫자를 나눗셈했을 때 자동 변환 된다.
            Number("123z") 에러가 발생한다.
        
            3. bool 변환
            일단, 비어있는 값들은 모두 false 처리가 된다.
            다른 값은 true이거나
            alert("hello"); -> true
            alert(""); -> false 스페이스로 공간이 있으면 true가 됩니다.
            undefined는 NaN이 아니라 0이다.

    7. Comparisons

    비교

                7. 비교
            두 문자열은 사전 순서로 비교한다.
            두 문자열이 같은 길이로 끝나면 길이가 긴 문자열이 더 크다.
            같은 문자의 대소문자 비교에서는 소문자가 내부 인코딩 테이블에서 더 큰 인덱스를 갖기 때문에 더 크다.
            엄격한 같음 연산자를 사용하고 싶은 경우에는 === 를 사용함.
            수학적으로 null==0은 아니지만 null >=0은 true 이다.
            
                5 > 4 → true
                "apple" > "pineapple" → false
                "2" > "12" → true // 사전 비교이기 때문에 첫번째 char을 비교하면 2가 크다.
                undefined == null → true
                undefined === null → false
                null == "\n0\n" → false typeof string
                null === +"\n0\n" → false typeof number 0

    8. Conditional branching if, "?"

    조건문

                
            물음표 연산자와 if 조건 연산자를 이용한는데, 명령문은 괄호 안의 조건을 평가하고 결과가 true이면 실행한다.
            let  year = prompt("Your birthday?"); 
            if (year == 2015) alert("true");
            둘 이상의 명령을 넣을 때엔 괄호로 묶어야 한다.
            
            bool 변환
            false - 숫자 0, null, "" , undefined, NaN 모두 
            
            else 절
            else 절은 if 절 이외의 조건에 경우
                if (year == 15){
                    alert("y");
                }else if (year > 15) { 
                    alert("f");
                }else{ 
                    alert("m")
                }
            같은 방법으로 ?를 이용하여 표현 가능
            let result = condition? true 값: false 값;
            let access = (age>18)? true : false;

            
            ?를 여러개 사용하고 싶은 경우
            let age = prompt('age?',8);
            let message = (age > 3) ? hi :
            (age >3) ? "hello":
            age > 100 ? "great":
            "oop";

        
            위 코드의 흐름을 보면 먼저 첫번째 문장 진위 여부를 판단하고, 
            false이면 다음 문장을 진위여부를 진단하면서 넘어갑니다.

    9. Logical operators

    논리 연산자

                OR || - 여러개의 조건은 첫번째 true 값을 찾는다. 
            And && - 첫번째 거짓값을 찾는다.
            And 연산자는 기본적으로 OR보다 우선순위가 높다.
            a && b || c && d == && (a && b) || ( c && d )
            let x =1 ; 
            ( x>0 ) && alert("greater"); 
            같은 의미인 문장이
            if( x>0 ) alert("greater");
            alert( alert(1) || 2 || alert(3) );
            alert 먼저 되고 그 다음 2를 반환하는 이유는 alert는 반환하고 나서 true가 아니라 undefined 된다.
            따라서 다시 true값을 찾아서 나간다.
